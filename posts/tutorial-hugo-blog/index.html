Build Static Website With Hugo and Github Pages

introduction

A static website is one with stable content where every user sees the exact same thing on each individual page. There are tools to build static website, such as wordpress, hugo, jekyll, etc. In this tutorial i want to explain how to use Hugo to build static website. After you build the static website, you need to host it online. There are many services that help you to host your website online, either paid or free, such as netlify, github pages, heroku, etc. In this tutorial, i want to explain how you to use github pages to host your website.

step 1 : install hugo

Installing hugo in linux or macos is straightforward , you can follow this instruction.

Installing hugo in windows is not that straightforward. First, you have to download the latest release of Hugo from their official download page, scroll down and download zip files for windows (example : hugo_0.99.0_Windows-64bit.zip). After that, extract the zip file to C:\Hugo\bin . Next, you need to set an environment variable. These are steps to do that :

  • open search bar in bottom left corner your screen
  • look for Edit the system environment variables and open it
  • in Advanced tab, click Environment Variables
  • in the top half box, search for Path and then double click on that.
  • click new and then type the path of extracted hugo file, in this case is C:\Hugo\bin and then click ok

To test the installation, open up your terminal and then type hugo version. If your installation is success, you should get back information about hugo version that has been installed.

step 2 : build your site locally

First, you should go to the directory where you want to save your website. Let say i want to build my website in C:\Git in folder mysite, i will open terminal and type :

cd C:\Git
hugo new site mysite

After that, the website is successfully built in C:\Git\mysite and you will see many files there, the three most important are content folder, themes folder, and config.toml. You write the content of your website in content folder, you choose theme for your website in themes folder, and you configure your website according to theme you used in config.toml.

Until this step, you can’t see your website yet because you have not chosen theme. So, now you should choose theme, in this tutorial i will use this theme as example.

From the root of your website folder, type :

git init
git submodule add https://github.com/zwbetz-gh/minimal-bootstrap-hugo-theme.git themes/minimal-bootstrap-hugo-theme

Next, copy the config.toml in C:\Git\mysite\themes\minimal-bootstrap-hugo-theme\exampleSite to config.toml in C:\Git\mysite. What you do here is you copy the configuration of exampleSite from minimal-bootstrap-hugo-theme to your hugo website folder. If you want advance configuration, you need to read documentation of the theme you use. Usually, a theme has exampleSite folder with its config.toml inside it.

Back to root of your website in terminal and then type hugo to build your website and then type hugo server to open your website locally. After you type hugo server there will be localhost address in your terminal, left click + ctrl to open it and your website will be open in browser.

step 3 : deploy to git pages

After you successfully run your website locally, next step is deploying your website online using git pages.

iIn the previous step, after you type hugo in root folder of your website, you will notice that there is new folder public appear. This is the rendered version of your website and we only need this folder and anything contain in it to used in github pages. Remember that when you update your website, you need to type hugo in root folder to update the rendered folder in public folder.

I hope you have already have github account in this moment. To create github pages for your website, first create new repository in github with repository name using your github username and don’t initiate with README.md. After your repository have been created, go back to your root directory of your website in your computer terminal and type :

cd public 
git init 
git add .
git remote add origin https://github.com/username/username.github.io
git commit -m "first commit" 
git push origin master

Remember to change username with your github username. After successfully committed, now finally you can access your website in username.github.io.
